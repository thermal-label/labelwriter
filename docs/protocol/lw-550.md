# LW 550 Raster Protocol

This page documents the wire protocol of DYMO LabelWriter **550** /
**550 Turbo** / **5XL** printers (`engine.protocol === 'lw-550'`). The
550 firmware is a clean break from the 450-family protocol — different
job structure, an explicit job header / trailer, per-label `ESC D`
header with a 12-byte preamble, and a 32-byte status reply with rich
NFC-derived media diagnostics. It is **not** wire-compatible with the
450 family; the encoder dispatches on `engine.protocol` and emits a
fundamentally different byte stream.

::: info Source
The byte sequences below are sourced from *LW 550 Technical Reference*
(Sanford, 2021). The PDF is the authoritative reference; this page
summarises what the driver actually emits and parses, with annotations
where the spec is ambiguous (e.g. `ESC L` parameter format, density
percent mapping). The reference document is **not** redistributed —
DYMO publishes it from their support site.
:::

## Genuine media is enforced in firmware

The 550 family validates every loaded label roll against an NFC tag
embedded in the spool. Counterfeit or third-party media is detected
during cassette insertion and surfaced as bay-status `10` in the
status reply. **Sending a valid job header does not bypass NFC
validation** — the check is hardware-side, independent of the wire
protocol. Plan around genuine DYMO consumables for any application
targeting these printers.

## Models and engines

| Family             | Head dots | Notes |
| ------------------ | --------: | ----- |
| LW 550 / 550 Turbo | 672       | Standard 56 mm-class head |
| LW 5XL             | 1248      | 4-inch wide head; same protocol |

`engine.protocol` is `lw-550` for all of the above.

## USB topology

Standard USB Printer-class device on VID `0x0922`. PIDs are listed on
the [Hardware](../hardware) page. The 550 status reply is **32 bytes**
(vs the 450's 1 byte) — `statusByteCount(device)` reports the correct
size to read after `ESC A`.

## Print job structure

```
(once per job)
  ESC s <jobId u32>          — start of print job
  ESC h | ESC i              — text vs graphics mode
  ESC C <duty u8>            — set print density (0..200 %)
  [ESC T <speed>]            — content type / speed mode (optional)

(per copy, label index 0..N-1)
  ESC n <index u32>          — label index
  ESC D <bpp> <align> <width u32> <height u32>
                             — start of label print data (12-byte header)
  <raster bytes>             — width × ceil(height × bpp / 8) bytes
                             — NO SYN prefix; rows are one after another
  ESC G  (between copies)
  ESC E  (last copy)         — feed to tear

ESC Q                        — end of print job (mandatory)
```

All multi-byte integers are little-endian.

::: warning Don't emit 450-style preamble
The 550 firmware does **not** accept `ESC @` / `ESC D <n>` /
`ESC L <lo> <hi>` as the start of a job. Any 450-style preamble
results in a job stream the 550 cannot parse and the status enum
transitions to `error`. Use the `encodeLabel(device, …)` helper —
it dispatches on `engine.protocol` and always emits the right shape.
:::

### `ESC s <jobId u32>` — start of print job (`1B 73 …`)

4-byte little-endian job ID, any value. The driver defaults to
`Date.now() & 0xFFFFFFFF`. The ID is echoed back in bytes 1–4 of every
status reply during the job, so the host can correlate.

### `ESC h` / `ESC i` — output mode (`1B 68` / `1B 69`)

`ESC h` (`0x1B 0x68`) selects text mode; `ESC i` (`0x1B 0x69`) selects
graphics. Same opcodes as the 450 family.

### `ESC C <duty>` — print density (`1B 43 nn`)

Single byte, range `0..200` (percent). Default is `100`. The driver's
family-level `Density` enum maps as:

| Density   | Duty (%) |
| --------- | -------: |
| `light`   | 70       |
| `medium`  | 85       |
| `normal`  | 100      |
| `high`    | 130      |

Pass an explicit number to `build550Density()` to bypass the mapping.

### `ESC T <speed>` — content type (`1B 74 nn`)

Optional. `0x10` = normal speed, `0x20` = high. Affects the print head
duty cycle. Omitted by default; the driver only emits it when the
caller explicitly sets `options.speed`.

### `ESC n <index u32>` — label index (`1B 6E …`)

Per-label header, sent before each label's `ESC D` block. 4-byte
little-endian index starting at `0` for the first label in the job.

### `ESC D` — start of label print data (12-byte header) (`1B 44 …`)

```
1B 44 <bpp> <align> <width0..3> <height0..3>
```

| Field   | Bytes | Meaning |
| ------- | ----: | ------- |
| `bpp`   | 1     | Bits per pixel; default `1` |
| `align` | 1     | Alignment mode; default `2` (bottom) |
| `width` | 4 LE  | Spec "Width" = number of raster lines (= feed direction = our `bitmap.heightPx`) |
| `height`| 4 LE  | Spec "Height" = dots per line (= head width = our `bitmap.widthPx`) |

::: warning Axis convention
The spec's `Width` is the feed direction (raster line count); `Height`
is the cross-head dot count. This is opposite to the natural reading
of "width × height" for a printed label — the spec p.12 diagram makes
it concrete: dot 0 sits at the bottom of the head and raster rows
advance with the feed. The driver hides this by accepting a head-aligned
`LabelBitmap` whose `widthPx` is the head dot count and `heightPx` is
the raster line count.
:::

### Raster payload

`width × ceil(height × bpp / 8)` bytes follow the `ESC D` header
**directly** — no `0x16` SYN prefix, no length byte, no row-by-row
framing. The 450 family's `0x16` / `0x17` framing does **not** apply.

`compress` is silently ignored on this protocol — the 550 raster
format does not carry the SYN/ETB framing the 450's RLE depended on.

### `ESC G` / `ESC E` — feed (`1B 47` / `1B 45`)

`ESC G` between labels (advances to the next print head position).
`ESC E` after the last label (advances to the tear position). Mismatch
produces a stalled job — make sure the **last** copy ends with `ESC E`
and the inter-copy feeds are `ESC G`.

### `ESC Q` — end of print job (`1B 51`)

Mandatory trailer. Releases the host's print lock and tells the
firmware the job stream is complete. The driver always emits this; if
you write your own encoder, do not skip it.

## Status — `ESC A <lock>` (`1B 41 nn`)

Three-byte form. The lock byte selects the request type:

| `lock` | Purpose |
| -----: | ------- |
| `0`    | Heartbeat / status query (default) |
| `1`    | Acquire print lock before sending a job |
| `2`    | Status query between labels in an active job |

Returns 32 bytes:

| Offset | Field                | Notes |
| -----: | -------------------- | ----- |
| 0      | Print status         | 0 idle, 1 printing, 2 error, 3 cancel, 4 woke, 5 lock-not-granted |
| 1–4    | Job ID               | Echoes the most recent `ESC s` |
| 5–6    | Label index          | u16, echoes the current `ESC n` |
| 7      | Reserved             | |
| 8      | Print-head status    | 0 ok, 1 overheated |
| 9      | Density              | u8 percent |
| 10     | Main bay status      | NFC media diagnostic — see table below |
| 11–22  | SKU (12-char ASCII)  | Identifies the loaded consumable |
| 23–26  | Error ID             | u32 LE; 0 = none |
| 27–28  | Labels remaining     | u16 LE |
| 29     | EPS status           | bit 0 = external power present |
| 30     | Head voltage         | 0 unknown / 1 ok / 2 low / 3 critical / 4 too-low |
| 31     | Reserved             | default `0xFF` |

`PRINT_STATUS_LOCK_NOT_GRANTED = 5` is the load-bearing one for the
lock-acquisition flow: it tells the caller "your `ESC A 1` did not
take the lock; another host is in charge."

### Main bay status (byte 10)

| Code | Meaning                                | Mapped to |
| ---: | -------------------------------------- | --------- |
|    0 | Unknown                                | (no signal) |
|    1 | Bay open, media presence unknown       | `cover_open` |
|    2 | No media present                       | `no_media` |
|    3 | Media not inserted properly            | `no_media` |
|    4 | Media present, status unknown          | (treat as loaded) |
|    5 | Media present, empty                   | `no_media` |
|    6 | Media present, critically low          | `low_media` |
|    7 | Media present, low                     | `low_media` |
|    8 | Media present, ok                      | `ready` |
|    9 | Media present, jammed                  | `paper_jam` |
|   10 | Media present, counterfeit             | `counterfeit_media` (NFC fail) |

## Auxiliary commands

### `ESC U` — Get SKU Info (`1B 55`)

Returns a **63-byte** NFC dump for the loaded roll: SKU number, magic
bytes, label dimensions, material, label / content colour, marker
geometry, label count, production date / time. The driver parses it
via `parseSkuInfo()` and surfaces a `detectedMedia` descriptor to the
adapter.

### `ESC V` — Get Engine Version (`1B 56`)

Returns a **34-byte** identity block: 16-char hardware version, FW
kind (`FWAP` application or `FWBL` bootloader), 4-char major/minor
versions, 4-char `MMYY` release date, and the USB PID (u16 LE,
bytes 32–33). Useful for verifying which firmware is talking before
attempting recovery.

### `ESC @` — restart engine (destructive)

Reboots the print engine. Loses any buffered data; only use when the
engine is wedged. The driver exposes `build550Restart()` but does not
call it from the recovery path — the soft path uses `ESC Q` instead.

### `ESC *` — factory reset (destructive)

`1B 2A`. Wipes user-tunable settings. Not exposed on the adapter; if
you need it, write the bytes via `transport.write()`.

### `ESC o <count>` — set label count (`1B 6F nn`)

Single-byte count `0..255`. Likely overrides the on-printer
remaining-labels counter. Exposed as a low-level builder.

## Recovery

Soft path: send `ESC Q` to release any pending job state and the host
print lock. Safe at any time. The driver's `build550Recovery()` emits
exactly `ESC Q`.

For a hard recovery, use `build550Restart()` (`ESC @`) — destructive,
loses buffered data.

## WebUSB

Same protocol as USB; the browser package opens Interface 0,
`transferOut` for the encoded job, `transferIn(32)` for the status
reply. WebUSB requires HTTPS or `localhost`.

## Porting checklist

- [ ] Use VID `0x0922`; the 550 family has its own PIDs (see
      [Hardware](../hardware)).
- [ ] **Don't** emit a 450-style preamble (`ESC @` / `ESC D` /
      `ESC L`). Always start with `ESC s <jobId u32>`.
- [ ] Per copy: `ESC n <index u32>` then `ESC D` 12-byte header then
      raster bytes (no SYN prefix).
- [ ] Last copy ends with `ESC E`; intermediate copies end with
      `ESC G`.
- [ ] Always emit `ESC Q` to close the job — without it the printer
      holds the host lock.
- [ ] Density is `ESC C <duty>` with `duty` in `0..200`. Default 100.
- [ ] Status request is **3 bytes** (`ESC A <lock>`) and returns
      **32 bytes**.
- [ ] Read main bay status (byte 10) for media diagnostics; SKU
      string lives in bytes 11–22.
- [ ] Watch for `printStatus == 5` (lock not granted) when calling
      `ESC A 1` to acquire the lock.
- [ ] Genuine NFC-validated DYMO consumables are required — there is
      no protocol-level bypass for counterfeit detection.
